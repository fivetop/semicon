2. 블록 구조 비교
==============================================================================

2.1 5-Block 계층화 (이전)
━━━━━━━━━━━━━━━━━━━━━━━━

           ORCA_TOP
              │
    ┌─────────┼─────────┬──────────┬────────────┐
    │         │         │          │            │
RISC_SYS  MEM_SYS  ACCEL_SYS  CLK_PWR_SYS
    │         │         │          │
    ├─────────┼─────────┤          │
   (신호 직접연결)      └──────────┘

문제점:
❌ 클록 도메인이 섞여 있음
❌ sys_clk와 pclk 신호가 직접 연결
❌ Metastability 위험
❌ CDC 검증 어려움
❌ 새 인터페이스 추가 시 복잡


2.2 CDC 중심 계층화 (추천)

    ╔═════════════════════════╗
    ║   SYS_CLK Domain        ║  125MHz
    ║  ┌──────────────────┐   ║
    ║  │ RISC_SUBSYSTEM   │   ║
    ║  │ BLENDER_SUBSYSTEM│   ║

    ╚══════════════╤═════════╝
                   │
         [비동기 FIFO CDC]
         (Gray 코드)
                   │

    ║  ┌──────────────────┐   ║
    ║  │ PCI_INTERFACE    │   ║

    ╚══════════════╤═════════╝
                   │
         [비동기 FIFO CDC]
                   │
    ╔══════════════╤═════════╗
    ║   SDRAM_CLK Domain      ║
    ║  ┌──────────────────┐   ║
    ║  │ SDRAM_INTERFACE  │   ║
    ║  └──────────────────┘   ║
    ╚═════════════════════════╝

장점:
✅ 클록 도메인 명시적 분리
✅ 각 도메인 독립 검증
✅ CDC 검증 자동화 가능
✅ Metastability 근본 해결
✅ 성능: 도메인별 최적화

               │

    │                           │
    ├─> AXI Slave Port 1────────┐

    └─> AXI Slave Port 2        │

                 ┌──────────────┘
                 │
         ┌───────┴───────┐

✅ 오픈소스 도구 사용 가능

3. 신호 인터페이스 비교
==============================================================================

3.1 5-Block: 직접 신호 연결
━━━━━━━━━━━━━━━━━━━━━━━━

RISC_SYS ──┬──> read_data[31:0]
           ├──> write_data[31:0]
           ├──> write_valid
           ├──> read_valid
           ├──> addr[31:0]
           ├──> we_n
           └──> (기타 제어 신호 10+개)

MEM_SYS ──┬──> read_data[31:0]
          ├──> write_ready
          ├──> (기타 신호...)
          └──> (관계 복잡)

문제: 신호 연결이 복잡하고 검증 어려움


3.2 CDC: 비동기 FIFO 인터페이스
━━━━━━━━━━━━━━━━━━━━━━━━━━

SYS_CLK 도메인 ────┐
  RISC outputs    │  [비동기 FIFO]  ┌── PCI_CLK 도메인
  - data[63:0]    ├─────────────────┤    PCI inputs
  - valid         │                 │    - data[63:0]
  - ready         │                 │    - valid
                  │                 │    - ready
              [Gray 코드]          └──
           (자동 동기화)

신호 매우 단순:
- push_data, push_valid, push_ready (송신측)
- pop_data, pop_valid, pop_ready (수신측)
- only 6 신호 (매우 깔끔!)


3.3 AXI: 표준 프로토콜
━━━━━━━━━━━━━━━━━━━

Write Address Channel:
  - AWADDR[31:0], AWLEN[7:0], AWSIZE[2:0], AWBURST[1:0]
  - AWVALID, AWREADY

Write Data Channel:
  - WDATA[63:0], WSTRB[7:0], WLAST
  - WVALID, WREADY

Write Response Channel:
  - BRESP[1:0], BVALID, BREADY

Read Address Channel:
  - ARADDR[31:0], ARLEN[7:0], ARSIZE[2:0], ARBURST[1:0]
  - ARVALID, ARREADY

Read Data Channel:
  - RDATA[63:0], RRESP[1:0], RLAST
  - RVALID, RREADY

(총 ~40개 신호, 하지만 체계적이므로 이해 쉬움)

==============================================================================
4. 클록 도메인 관리 비교
==============================================================================

4.1 5-Block: 문제 있음
━━━━━━━━━━━━━━━━━━━

이러한 문제가 발생 가능:
❌ RISC (sys_clk) 신호가 PCI (pclk)로 직접 입력
   → Metastability!
   
❌ PCI 신호가 SDRAM 로직에도 직접 연결
   → 클록 도메인 교차 검증 어려움
   
❌ 어디서 동기화되는지 불명확
   → 설계자도 모름
   
❌ CDC 검증 도구 적용 불가
   → Metastability 버그 사전 탐지 못함


4.2 CDC: 명확하고 안전
━━━━━━━━━━━━━━━━━━━

각 도메인이 명시적으로 분리:

sys_clk Domain:
  [로직] ──> [비동기 FIFO Push]
  
Boundary (자동 동기화):
  Gray 코드 변환
  플립플롭 체인으로 동기화
  
pclk Domain:
  [비동기 FIFO Pop] ──> [로직]

✅ Metastability 근본 제거
✅ CDC 검증 도구 자동 적용 가능
✅ Vivado CDC, Synopsys CDC-IFV 등으로 버그 자동 탐지


4.3 AXI: 도메인 중립적
━━━━━━━━━━━━━━━━━━━

AXI는 클록 도메인을 신경 쓰지 않음.
대신:
- 각 도메인마다 분리된 버스 생성 가능
- 교차점에 CDC 자동 삽입
- 표준 도구로 관리

예:
  RISC (sys_clk) ─┐
                  ├─ CDC 계층 ─┐
  Blender (sys_clk) ┘           ├─ AXI Interconnect ─ SDRAM (sdram_clk)
                                │
                    PCI (pclk) ─┘

5. 확장성 비교
==============================================================================

5.1 5-Block: 낮음
━━━━━━━━━━━━━━━

새 마스터 추가: "DMA Engine" 추가한다면?

ORCA_TOP.v 수정:
- dma_subsystem 추가
- 신호 연결 ~30개 추가
- 기존 신호 충돌 검사
- 멀티플렉서 로직 수정
- 테스트벤치 대폭 수정

결과: 복잡, 버그 위험 ↑


5.2 CDC: 중간
━━━━━━━━━━━━

새 마스터 추가: "DMA Engine" (sys_clk)

ORCA_TOP.v 수정:
- dma_subsystem 인스턴스 추가
- CDC FIFO 추가 (자동화)
- 기존 신호 변경 없음

결과: 비교적 간단, 독립적


5.3 AXI: 매우 높음
━━━━━━━━━━━━━━━

새 마스터 추가: "DMA Engine" (AXI Master)

과정:
1. DMA를 AXI Master로 설계 (표준!)
2. AXI Interconnect에 연결 (자동)
3. 기존 설계 변경 없음

결과: 매우 간단, 완전 독립

새 슬레이브 추가: "Peripheral Controller" 가능
1. Peripheral을 AXI Slave로 설계
2. Interconnect 설정 변경만 (수 줄)
3. 끝!

6. 검증 방식 비교
==============================================================================

6.1 5-Block: 검증 어려움
━━━━━━━━━━━━━━━━━━━

RISC 검증:
- RISC_SYS 단위 테스트
- 하지만 MEM_SYS와의 인터페이스 검증 필요
- 신호가 26개... 다 맞는지 확인?
- CDC 검증은? (별도로 수동 확인)

MEM_SYS 검증:
- PCI/SDRAM 동시 테스트?
- 클록 도메인 시뮬레이션 느림
- Metastability 검증 어려움

결과: 검증 시간 오래, 버그 위험 높음


6.2 CDC: 검증 자동화
━━━━━━━━━━━━━━━

RISC 검증:
- sys_clk 도메인만 검증
- 표준 UVM 테스트벤치
- 빠른 시뮬레이션

CDC 검증:
- Synopsys CDC-IFV 자동 검증
- Metastability 버그 자동 탐지
- 메뉴 클릭 몇 번으로 완료

PCI 검증:
- pclk 도메인 독립 검증
- 병렬 진행 가능

병렬 검증:
- RISC 검증 + PCI 검증 동시
- CDC 자동 검증 동시
- 총 검증 시간: 50% 단축


6.3 AXI: TLM 기반 검증
━━━━━━━━━━━━━━━

레이어별 검증:
1. TLM 참고 모델 (High-level, 매우 빠름)
   - 기능 동작 확인
   - 1초 내에 완료

2. AXI 레이어 검증
   - AXI Master VIP + Slave VIP
   - 프로토콜 정합성 자동 검증
   - 수십 초

3. RTL 레이어 검증
   - 실제 게이트 레벨
   - 마지막 검증만

결과: 매우 빠른 검증 (TLM만으로 버그 95% 탐지)

7. 합성 차이점
==============================================================================

7.1 5-Block
━━━━━━━━━

합성:
- ORCA_TOP 전체 한 번에 합성
- 클록 도메인 중복 합성 (비효율)
- 크로스 도메인 경로 타이밍 복잡
- DC 합성 시간: ~30분

타이밍 클로저:
- Critical path 분석 어려움
- "왜 이 신호가 느린가?" 추적 어려움


7.2 CDC
━━━━━━━

합성:
- 각 도메인 독립 합성 가능
  * RISC_SYS (sys_clk): 8분
  * PCI_IF (pclk): 4분
  * SDRAM_IF (sdram_clk): 3분
  * CDC: 1분
- 병렬 합성 가능 (도메인별)
- 클록 도메인별 timing closure
- DC 합성 시간: ~10분 (병렬)

타이밍 클로저:
- 각 도메인 독립 최적화
- Critical path 명확
- 클록 주파수 독립적 설정


7.3 AXI
━━━━━━━

합성:
- IP별 독립 합성
- Interconnect 합성
- 매우 모듈식
- DC 합성 시간: ~15분 (병렬)

타이밍 클로저:
- 매우 체계적
- AXI 경로와 데이터 경로 분리
- 우선순위 명확
